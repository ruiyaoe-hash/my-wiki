# 仓库地图 / Repo Map

> 一页看懂整个仓库。每个文件夹三句话：干什么、给谁看、删了会怎样。
> **规矩：新增文件夹必须先在本页登记，不登记不准加。**（2026-07-19 立）
> 本页按信息流组织：外部信息从左边进来，经过加工与质检，变成可信资产，再外化出去。

## 信息流主线（价值所在）

```
搜索 → 整合 → 吸收 → 分发 → 外化
（质量层贯穿全程：入库标准 + 巡检 + 核实标注）
```

### 搜索（把外部信息找回来）
- `scripts/ingest.py` — 入库脚本，抓取 URL/本地文件 / 给机器跑 / 删了可用 `agent-runtime ingest` 替代
- `source/original/` — 抓回的原文存档 / 给机器和追溯用 / 删了知识页失去证据链，不能删

### 整合（加工成结构化知识）
- `source/summaries/` — 原文的结构化摘要 / 给机器和追溯用 / 删了可从原文重新生成
- `executor/` — 协议执行器，所有加工动作的执行者 / 给机器跑 / 删了系统瘫痪，不能删

### 吸收（变成可信资产）
- `知识库/` — 知识页（给人读的 Markdown，Obsidian 里直接看）/ 给人看 / 删了项目失去内容本体，不能删
- `knowledge/` — 每页一份的机器元数据 sidecar + metadata-schema.json / 给机器读 / 删了可用 `scripts/rebuild_sidecars.py` 重建
- `memory/` — 五层记忆存储（L0-L4）+ sessions 归档 / 给机器跑 / 删了系统失忆
- `index.md` — 知识库索引（本地个人文件，不进仓库）/ 给人看 / 删了可重建

### 分发（提取运用）
- `agents/cli.py` — 结构化命令层，`agent-runtime` 各子命令入口 / 给人和机器用 / 删了回到散落脚本时代
- `console/` — 网页控制台（按钮 + 健康总览，只监听本机）/ 给人看 / 删了只能用命令行
- `reports/` — 巡检报告输出（不进仓库）/ 给人看 / 删了重新跑 check 即可
- `agents/wiki_agent.py` — 旧版循环入口（保留兼容）/ 给机器跑 / 删了影响旧脚本

### 外化（价值输出）
- `工作台/产出/` — digest 页等对外产出（本地，不进仓库）/ 给人看 / 删了丢产出记录
- 未来：MCP 只读接口、发布渠道（规划中，未建）

### 质量层（贯穿全程）
- `docs/ingest-standard-v0.1.md` — 入库标准：来源分级 + 核实标注 + 证伪留痕 / 给人看 / 删了质检失去制度
- `protocol/check.json` — 巡检协议（stale/断链）/ 给机器跑 / 删了失去自动体检
- `state/` — 运行时状态（会话、任务、执行计数）/ 给机器跑 / 删了可重建但丢历史

## 运行支撑层（系统的发动机，平时不用看）
