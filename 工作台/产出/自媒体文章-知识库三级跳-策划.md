 ## 文章主线
 
 从卡帕西的 LLM Wiki 启发，到谷歌的开源规范 OKF，再到清华的开源框架 OpenRath——三件事背后是同一个动作：**解耦**。把不该绑在一起的拆开，让该独立的独立。
 
 ## 三次解耦
 
 1. **卡帕西的第一次解耦：知识 vs 检索**：LLM Wiki 模式——LLM Wiki 把知识从每次查询重新检索里拆出来
 2. **谷歌的第二次解耦：格式 vs 平台**：OKF v0.1——OKF 把知识格式从平台私有 API 里拆出来
 3. **清华的第三次解耦：状态 vs Agent**：OpenRath——OpenRath 把 Agent 的状态从 Agent 自己脑子里拆出来——Agent 是水管，Session 是水
 4. **还差第四次解耦：生产 vs 存储**（独特视角）：OKF 管"知识怎么存"，OpenRath 管"知识怎么产"，两者之间的自动桥接——Agent 跑完 Session 自动写入 OKF 包——目前开源社区还没人做——这是我们一起发现的空白地带
 
 ## 对读者的价值（大白话）
 
 - 个人：如何用 OKF 格式管理知识，让 AI 真正帮你积累
 - 组织：如何用 OpenRath 的 Session 机制组织多 Agent 协作
 - 行业视角：谷歌和清华同时在推的事，方向已经明朗
