# Mem0 记忆层

> 来源：mem0ai/mem0 (59,571 stars) · Y Combinator S24 · [mem0.ai](https://mem0.ai)

Mem0 ("mem-zero") 是一个面向 AI Agent 的智能记忆层,通过在底层提供统一的记忆 API,让 Agent 具备记住用户偏好、适应个体需求、持续学习的能力。

---

| Benchmark | 旧算法 | 新算法 | Tokens | 延迟 p50 |
|-----------|--------|--------|--------|----------|
| LoCoMo | 71.4 | **91.6** | 7.0K | 0.88s |

### 算法核心设计

1. **单次 ADD-only 提取**——每次只做一次 LLM 调用,只添加新记忆,不做 UPDATE/DELETE。记忆累积而非覆盖。
2. **Agent 生成的事实与用户输入同权**——Agent 确认的操作信息获得同等存储权重。
3. **实体链接**——实体被提取、嵌入、跨记忆链接,用于检索增强。
4. **多信号检索**——语义 + BM25 关键词 + 实体匹配,三条线并行打分后融合。
5. **时间推理**——时间感知检索,能区分"当前状态""历史事件""未来计划",对"我的偏好变了吗"类问题给出正确的时效性答案。

### 与你 Wiki 的对比

| 维度 | Mem0 | 当前 Wiki |
|------|------|-----------|
| 写入方式 | ADD-only,自动提取 | 手动/Agent 单次写入 |
| 检索 | 语义+关键词+实体三线融合 | index.md → 链接遍历 |
| 实体管理 | 自动提取+嵌入+链接 | 手动 wikilink |
| 时间感知 | 原生支持 | log.md 手动记录 |
| 规模 | 10M token 级别 | 百页级别 |

### 可借鉴的设计

- **ADD-only 模式**：只追加不覆盖,信息不会丢失。适合 Agent 自动 commit 场景。
- **实体链接**：自动识别已存在的实体,新记忆自动链接过去——这正是 Wiki 交叉链接的自动化版本。
- **多信号检索融合**：纯语义检索不够精准,纯 BM25 丢失语义,融合后显著提升召回。
- **时间推理**：让检索知道"现在的我"和"过去的我"的区别。
