## 研究来源

加州大学伯克利 2025 年 NeurIPS 论文：分析 5 个开源多 Agent 系统（MetaGPT、AutoGen 等）1200+ 条执行记录。

## 核心发现

| 指标 | 数据 |
|------|------|
| 系统失败率范围 | 41% ~ 86.7% |
| 失败模式总数 | 14 种 |
| 最大失败来源 | 协调崩溃（36.9%） |
| 错误放大倍数 | 17.2 倍（Google DeepMind） |

## 14 种失败模式完整列表

1. **协调崩溃** — Agent 间沟通衔接失效，是最大单类失败来源（36.9%）
2. **信息传递断层** — 关键信息在 Agent 传递链中丢失
3. **角色混淆** — Agent 越权操作或职责边界不清
4. **重复劳动** — 多个 Agent 独立执行相同或高度重叠的任务
5. **死循环** — Agent 陷入无限交互，无法收敛到结果
6. **上下文膨胀** — 对话历史过长，超出模型上下文窗口或推理退化
7. **工具误用** — Agent 调用工具时参数错误、工具选择不当、或误解工具输出
8. **目标漂移** — 多轮交互后 Agent 偏离了原始任务目标
9. **过早终止** — Agent 在任务未完成时错误地判定为已完成
10. **幻觉传播** — 一个 Agent 产生的事实错误被下游 Agent 当作正确输入继续使用
11. **权限混乱** — Agent 在缺少适当权限的情况下执行敏感操作
12. **资源竞争** — 多个 Agent 同时争抢同一资源（如文件写入、API 配额）
13. **记忆不一致** — Agent 的长期记忆与实际状态出现偏差
14. **路由错误** — 任务被分配给不擅长的 Agent，导致低质量输出

## 失败模式与 Session 中心化的关系

Session 中心化设计（如 OpenRath）直接针对其中多种失败模式：

| 失败模式 | Session 中心化的解法 |
|------|------|
| 协调崩溃 / 信息传递断层 | 所有 Agent 共享同一 Session，取消"传话"环节 |
| 角色混淆 | Lineage 记录每条结论的来源 Agent，职责清晰可追溯 |
| 重复劳动 | Session Graph 的 fork/merge 让并行分支可见，避免重复 |
| 幻觉传播 | Lineage 可追溯到原始数据源，交叉验证 |
| 上下文膨胀 | Compressor 压缩 + fork/detach 清理无用分支 |
| 路由错误 | Selector 动态路由，根据 Session 状态实时决策 |

## 启示

设计 Agent 系统不能只用传统软件的"规则+边界"模式，需要适应涌现行为。Session 中心化提供了一种思路：不试图控制每个 Agent 的行为，而是控制它们共享的状态——状态对了，协作就对了。
