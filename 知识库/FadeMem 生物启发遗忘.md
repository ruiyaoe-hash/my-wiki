# FadeMem 生物启发遗忘

> 来源：[arXiv 2601.18642](https://arxiv.org/abs/2601.18642) · 2026 年 1 月 · Lei Wei, Xiao Peng, Xu Dong, Niantao Xie, Bin Wang

FadeMem 是一套借鉴人类记忆遗忘机制的 Agent 记忆架构。它的核心立场是：**当前 AI 系统的记忆策略是二元的——要么全留，要么全丢**。人类记忆在「保留」和「遗忘」之间有一条平滑的衰减曲线，FadeMem 要做的就是把这条曲线搬到 Agent 里。

---

## 核心机制

### 双重记忆层级 (Dual-Layer Memory Hierarchy)

FadeMem 将记忆分为两层，每层有不同的衰减速率：

| 层级 | 作用 | 衰减特征 |
|------|------|----------|
| **快层 (Fast)** | 临时交互信息 | 高衰减率，短期优先淘汰 |
| **慢层 (Slow)** | 结构化长期知识 | 低衰减率，受语义相关性保护 |

### 自适应指数衰减 (Adaptive Exponential Decay)

每条记忆的保留程度由三个因子共同调制：

1. **语义相关性** — 与当前任务有多相关
2. **访问频率** — 被检索了多少次
3. **时间模式** — 多久前写入的

不是简单的「过 N 天就删」，而是三个因子乘积后的连续衰减函数。相关的、常用的、最近的 → 衰减慢；无关的、冷门的、久远的 → 衰减快。

### LLM 驱动的冲突消解与记忆融合

当两条记忆之间存在矛盾或重叠时，不靠规则硬裁——交给 LLM 做语义级判断，合并相关信息、丢弃冗余细节。

## 基准测试结果

| 基准 | 评估维度 | 结果 |
|------|----------|------|
| Multi-Session Chat | 多轮跨会话对话 | 推理 + 检索性能优于 baseline |
| LoCoMo | 长期上下文建模 | 45% 存储缩减下性能不降 |
| LTI-Bench | 长期交互基准 | 多跳推理表现优异 |

**关键数字：45% 存储缩减。** 不是压缩算法，而是通过「主动遗忘不重要信息」做到的。

---

## 与其他方案对比

| 方案 | 策略 | 遗忘机制 |
|------|------|----------|
| Mem0 | ADD-only，不删不改 | 无主动遗忘，靠检索时过滤 |
| MemGPT/Letta | 虚拟上下文管理（OS 式换页） | 基于上下文窗口的被动淘汰 |
| HippoRAG | 海马体式联想检索 | 无主动遗忘，重检索 |
| **FadeMem** | **生物启发主动遗忘** | **自适应指数衰减 + 记忆融合** |

FadeMem 是已知第一个把「遗忘」作为一等设计目标（而非副作用）的 Agent 记忆方案。

## 与你的知识库

FadeMem 验证了你之前文章里提的方向：AI 记忆的未来不是「存得更多」，而是「忘得更聪明」。你的 [[AI记忆的遗忘曲线]] 从人类认知出发推导了这个逻辑，FadeMem 从工程上实现了它。

当前知识库的内存管理是纯 Markdown 文件 + Git 版本控制的静态模式——不会主动遗忘。如果未来要做一个「活的」知识管理系统，FadeMem 的三因子衰减模型可以作为记忆生命周期的设计参照。
