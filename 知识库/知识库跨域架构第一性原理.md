# 知识库跨域架构第一性原理

当知识库从单一领域扩展到多领域时，需要一个不随域数量增长的架构方案。以下从第一性原理推导，并经四大学派交叉验证。

## 五条公理

### 公理 1：知识的价值在连接，不在存储

一个知识系统的首要价值不在于它存了多少东西，而在于它能在不同概念之间建立多少有意义的连接。跨域连接——比如"CoALA 认知架构可以指导 TPIS Pro 的 Agent 记忆设计"——恰恰是最有价值的洞察类型。每增加一个独立 vault，这类连接的发现成本翻倍。

**推论**：多 vault 方案直接降低系统价值。

### 公理 2：分类是下游行为，不是上游结构

你无法在第一次接触一个概念时就正确归类——归类需要理解，理解需要时间。用文件夹做顶层结构，等于用最不成熟的理解去锁定知识的位置。"AI 工程"和"文旅策划"今天看起来是两个域，但它们正在融合成"用 Agent 系统做文旅策划的方法论"。文件夹结构会阻碍这种融合。

**推论**：用轻量标签（frontmatter domain）代替文件夹层级做域标记。

### 公理 3：检索场景决定组织方式

你不会说"我想随便看点文旅的东西"然后去翻文旅文件夹。真实的检索场景是任务驱动的："TPIS Pro L3 阶段用什么框架做竞品扫描？"这个问题的答案可能在 AI 工程域（Agent 框架对比）、文旅域（TPIS Pro 流程）、或两者之间。按域分库意味着这个问题需要查两个 vault。

**推论**：组织方式应匹配检索方式。MOC 导航 + 全文搜索 + wikilink 跳转 > 文件夹浏览。

### 公理 4：复杂度预算是有限的

每多一个 vault，就多一套 AGENTS.md、log.md、hot.md、ingest 管道。当这些基础设施版本漂移时，维护成本不是加法而是乘法。人的精力不是无限的——把预算花在连接知识上，而不是维护基础设施上。

**推论**：单 vault 的维护成本是 O(1)，多 vault 是 O(n)。

### 公理 5：一个大脑，一个知识库

你的大脑不会给"文旅策划知识"和"AI 工程知识"分配不同的神经元区域。知识的涌现发生在边界上。人为切割只会制造更多边界。

**推论**：知识库应该模拟大脑的互联结构，而非文件柜的隔离结构。

## 四大学派验证

| 学派 | 核心主张 | 对单 vault 的态度 |
|------|---------|-----------------|
| Zettelkasten (Luhmann) | 一个人一生只用一个卡片盒 | 明确支持 |
| PARA (Tiago Forte) | 按行动性组织，不是学科 | 隐含支持（跨域是默认状态） |
| Evergreen Notes (Matuschak) | 关联本体优于层级分类 | 明确支持 |
| MOC/LYT (Nick Milo) | MOC 提供导航，不限制访问 | 明确支持 |

**四派无一例外，全部反对按领域做顶层分割。**

## 推荐架构

```
单 vault
├── 知识库/（扁平 wikilink 网络）
│   ├── Zettelkasten 方法.md
│   ├── CoALA 认知架构.md
│   ├── TPIS Pro L3 竞品扫描.md
│   └── 🏠 AI工程.md（域 MOC）
│   └── 🏠 文旅策划.md（域 MOC）
├── 原始资料/（扁平，对应知识页）
├── _raw/（原始存档）
├── index.md（域路由器，不含页面清单）
├── hot.md（加"本次会话领域"字段）
├── log.md（每条记录加域标记）
├── milestones.md（每条记录加域标记）
└── AGENTS.md（域无关，不变）
```

## 与多 Vault 方案的量化对比

| 维度 | 多 Vault | 单 Vault + MOC |
|------|---------|---------------|
| 跨域 wikilink | 不可用 | 畅通 |
| 基础设施 | N 套，漂移风险 | 1 套 |
| 图谱纯净化 | 天然实现 | frontmatter domain 过滤等价实现 |
| 摄入管道 | N 套独立维护 | 1 套，domain 字段分流 |
| 知识涌现 | 边界被阻断 | 边界产生价值 |
