# Memory as Metabolism 记忆代谢

> 来源：[arXiv 2604.12034](https://arxiv.org/abs/2604.12034)（2026 年 4 月）
> 定位：AI 伴随知识系统的设计框架

## 核心理念

论文将记忆管理比作新陈代谢，而非数据库存储。记忆不是静态地"存进去等查"，而是经历摄取、消化、吸收、排泄的完整生命过程。

## 五步保留策略

1. **TRIAGE（分诊）**：摄入时浅过滤，在流式实时层拒绝明显的垃圾信息
2. **CONTEXTUALIZE（情境化）**：将外部信息与用户的工作语境深度对齐
3. **DECAY（衰减）**：基于最近调用 × 访问频率 × 结构重要性的加权衰减
4. **CONSOLIDATE（巩固）**：批处理整合，对条目进行相互评分和冲突消解
5. **AUDIT（审计）**：对最高重力条目做压力测试，防止教条固化

## 两个关键创新概念

### 记忆重力 (Memory Gravity)

被多个其他条目引用的节点是"承重"节点。移除它们会导致知识体系崩塌。因此衰减算法必须保护高重力条目，不能简单地按时间淘汰。

### 少数派假设保留 (Minority-Hypothesis Retention)

矛盾证据在缓冲区中持续积累压力。三个互相支持的条目可以挑战一个高重力的主流解释，防止库恩式的范式僵化。

## 镜像 vs 补偿原则

伴随记忆系统应该在两个维度上采取不同策略：
- **镜像 (Mirror)** 用户的操作维度：词汇、语境连续性、承重结构
- **补偿 (Compensate)** 用户的认知失败模式：固守成规、确认偏误、单一文化崩塌、矛盾压制
