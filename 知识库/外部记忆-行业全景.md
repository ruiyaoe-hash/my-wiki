## 三、前沿框架对比

### Mem0（59k stars, YC S24）
- Universal memory layer for AI Agents
- 2026 年 4 月新算法：单次 ADD-only 提取，不做 UPDATE/DELETE
- 实体链接 + 多信号检索（语义 + BM25 + 实体匹配）
- 时间推理：区分当前状态、历史事件、未来计划
- 基准测试：LoCoMo 91.6、LongMemEval 94.8、BEAM(1M) 64.1
- 定位：记忆即基础设施

### HippoRAG 2（3.8k stars, NeurIPS 24 -> ICML 25）
- 从 RAG 到 Memory 的范式升级
- 受海马体记忆机制启发，增强 LLM 的联想检索和意义建构能力
- 比 GraphRAG、RAPTOR、LightRAG 更省资源
- 核心贡献：非参数持续学习

### Letta / MemGPT（2.8k stars）
- 状态化 Agent：agents that are more like people than tools
- Agent 有记忆、有身份、有经验感知
- 自我编辑记忆（memory blocks）、技能学习、MemFS（Git 管理上下文）
- 桌面 App + CLI + 浏览器 + 消息集成
- 关键概念：inner/outer monologue 架构

### Agent_Memory_Techniques（557 stars）
- 30 个可运行 Jupyter Notebook，覆盖全部记忆模式
- 六大分类：短期 / 长期 / 认知架构 / 检索路由 / 框架 / 评估生产
- 明确提及 Anthropic 的 7 层记忆模型（2026 年 3 月）

### OpenCove（1.5k stars）
- 为 AI Agent 设计的无限画布工作空间
- 空间化上下文：Agent 会话 + 终端 + 任务 + 笔记放在同一画布
- 解决"线性失忆"问题——上下文不再藏在长对话历史里
- 直接启发了"带 UI 的知识管理系统"的产品形态

## 四、学术前沿

### CoALA：Cognitive Architectures for Language Agents
- 将模块化记忆组件定义为语言 Agent 架构的核心要件
- Agent = 记忆模块 + 行动空间 + 决策过程
- 记忆是第一公民，不是附加功能

### MMP：Mesh Memory Protocol (arXiv:2604.19540)
- 多 Agent 跨 Session 认知状态共享的形式化协议
- 核心论断："memory that survives session restarts is relevant because of how it was stored, not how it is retrieved"
- CAT7 七字段 schema + SVAF 锚点评估 + 跨 Agent lineage

## 六、综合判断

1. **Wiki 范式不是过渡方案。** 从 Karpathy 的概念到 OKF 规范到 Mem0/Letta 的工程化，多路人马独立推进同一方向。
2. **模型能力增强不构成替代。** 上下文窗口解决存储，Wiki 解决认知结构，两者正交。
3. **记忆正在从功能变成基础设施层。** Mem0 的 Universal memory layer 定位、Anthropic 的 7 层记忆模型、学术界的架构级论证，都指向记忆是第一公民。
4. **"带 UI 的知识管理系统"有明确的产品参照。** OpenCove 展示了 Agent + 空间化知识画布的可行形态；Obsidian + Dataview + Graph View 是另一个成熟参照。
5. **当前 Wiki 的改进方向**：
   - 引入 Mem0 式的实体链接和多信号检索
   - 参照 HippoRAG 的联想检索和持续学习
   - 借鉴 Letta 的自我编辑记忆（Agent 主动优化自己的上下文）
   - 参照 Agent_Memory_Techniques 的 30 种技术分类，补齐缺少的模式
   - 考虑 OpenCove 式的空间化 UI 方向
