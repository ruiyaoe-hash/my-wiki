# 踩坑集

> 每次 AI 走错方向、重复试错、输出不符合预期时，花 30 秒记一条。
> 目标：积累 20 条后做一次模式分析，找出最常见的失败类型和改进方向。

## 为什么要有这个文件

翁荔（Lilian Weng）在《Harness Engineering for Self-Improvement》中指出：负面结果被系统性忽视，是当前 AI 自我提升系统的核心瓶颈之一。AI 的训练数据里成功案例远多于失败，导致 AI 不擅长判断"该放弃"或"这条路走不通"。

这个文件就是为了补上这个缺口——把失败记录下来，让以后的我（和以后的 AI）能从中学习。

## 记录格式

每条记录用这个模板：

```
## [日期] 一句话标题
- 现象：发生了什么问题（用户看到的效果）
- 原因：根因分析（如果知道）
- 怎么办：修复方式 / 以后怎么绕
- 标签：#工具 #协议 #编码 #模型 #其他
- 关联：log.md 或会话归档里的对应记录（如有）
```

### 模板示例

## [2026-07-08] apply_patch 对中文路径文件经常匹配失败
- 现象：用 apply_patch 编辑含中文内容的 index.md 时，总是"Failed to find context"或"invalid hunk"
- 原因：apply_patch 的上下文匹配对多字节字符处理不稳定
- 怎么办：编辑中文文件时不用 apply_patch，改用 Python 脚本读写整个文件。能力地图已标注此限制
- 标签：#工具 #编码
- 关联：本次会话多次遇到
