# PARA 方法 — Tiago Forte 原文摘要

来自 Forte Labs 官方博客，Tiago Forte 是《Building a Second Brain》作者，PARA 是他提出的最广泛使用的数字信息组织系统。

## 四类文件夹

1. **Projects** — 有明确目标和截止日期的短期任务（写完报告、装修浴室）
2. **Areas** — 需持续关注的长期责任域（健康、财务、写作）
3. **Resources** — 感兴趣并正在学习的主题（咖啡、平面设计、记笔记）
4. **Archives** — 以上三类中不再活跃但需保留的条目

## 核心原则

**按项目和目标组织，不按学科/领域。** Forte 明确指出学校教我们"按学科分类（数学、历史、化学）"是错的，成年人不应沿用。

**简单性法则**：如果你的组织系统和你的人生一样复杂，维护它的代价会剥夺你生活的时间。

## 与 Zettelkasten 的共识

两者都反对按学科/主题做顶层分类。PARA 按"行动性"组织（这个项目需要什么？），Zettelkasten 按"概念连接"组织（这个想法和什么相关？）。方向不同，但都拒斥领域文件夹。

## 与我们 Wiki 的对应

PARA 是 workflow 系统，我们是 knowledge curation 系统。概念上不完全对应，但可以理解为：
- 我们按"知识域 MOC → 概念页"组织，PARA 按"项目 → 资源"组织
- 共通点：顶层不由学科分类决定
