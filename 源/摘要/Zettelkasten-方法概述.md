# Zettelkasten 方法官方概述

来自 zettelkasten.de 官方 Overview 页面，由 Sascha Fast 和 Christian Tietze 维护——这是 Luhmann Zettelkasten 方法在英文世界最权威的入口。

## 核心主张

**只用一个 Zettelkasten。** 官方 FAQ 第一条："How many Zettelkästen should I have? The answer is, most likely, only one for the duration of your life."

**不要用分类，用标签。** 反复强调：Don't use categories. Use tags instead.

**链接优先于搜索。** Set links between notes. Full-text search on its own provides not enough information.

## 三层架构

- **Bottom Layer**: Content（内容笔记）
- **Middle Layer**: Structure Notes（结构笔记，类似 MOC）
- **Top Layer**: Main Structure Notes and Double Hashes（主结构笔记 + 特殊标签）

## 关键原则

- **原子性**：一条笔记一个主题，给 ID，限制内容范围
- **连接性**：链接 vs 笔记序列，不同种类的笔记间联系
- **收藏家谬误**：仅仅收集信息不增加知识，必须加工

## 证据层级

数据描述(Data description) → 解释(Interpretation) → 综合(Synthesis)
