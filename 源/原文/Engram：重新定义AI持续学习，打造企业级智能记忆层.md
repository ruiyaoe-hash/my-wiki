# Engram：重新定义AI持续学习，打造企业级智能记忆层

> 来源: 得到大脑 · 创建: 2026-06-24

### **🚀 公司概况与融资背景**

**核心定位**  
Engram是一家聚焦AI持续学习能力的初创公司，致力于构建能从用户日常资料、对话和经验中持续学习的AI系统，解决当前模型"懂互联网却不懂企业"的关键痛点。

**融资与投资方**  
- **融资规模**：9800万美元  
- **投资方**：General Catalyst、Kleiner Perkins、Sequoia等头部风投机构  
- **顾问团队**：包括Assaf Rappaport、**Andrej Karpathy**（前特斯拉AI负责人）和**Pieter Abbeel**（UC Berkeley教授）  

**行业反响**  
- Andrej Karpathy公开祝贺："Congrats on the launch!!"  
- Jason Wei（前Google DeepMind研究员）评价："cool vision, congrats on the launch!"

### **🎯 核心问题与解决方案**

**当前AI的局限性**  
| 问题类型 | 具体表现 |
| :------- | :------- |
| **知识断层** | 模型仅掌握训练数据，对企业内部项目决策、历史取舍、团队讨论等私有知识不熟悉 |
| **效率低下** | 需反复提供上下文，每次对话需重新理解，成本高且易出错 |
| **记忆缺失** | 对话结束后遗忘信息，无法形成长期知识积累 |

**Engram的创新路径**  
- **核心技术**：构建AI持续学习的"**记忆层**"，将计算前移至预学习阶段  
- **实现方式**：提前学习GitHub、Slack、Notion等企业数据源，而非临时检索  
- **更新频率**：目前为每日学习，目标提升至每小时，最终实现**每分钟更新**  
- **技术目标**：开发统一训练算法，吸收任意规模、任意形式数据，实现模型持续进化

### **💼 产品布局与应用场景**

**首个产品形态**  
面向Agent的API服务，聚焦大型共享知识工作区。

**早期合作案例**  
| 合作方 | 应用场景 | 核心价值 |
| :----- | :------- | :------- |
| **Notion** | 构建理解大型Notion工作空间的Custom Agents | 解决知识密度高、上下文复杂的检索难题 |
| **Harvey** | 律所和企业知识场景 | 提升专业领域知识的长期消化能力 |
| **Microsoft** | M365定制化Agent试点 | 优化办公环境中的智能交互效率 |

### **👨‍👩‍👧‍👦 团队背景与技术基因**

**核心成员**  
包括Dan Biderman、Sabri Eyuboglu、**Jessy Lin**、**Jack Morris**等，团队背景高度集中在**持续学习**和**模型记忆**领域。

**研究方向**  
- 持续学习算法设计  
- 上下文压缩技术  
- 检索增强（RAG）优化  
- LoRA（低秩适应）应用  
- 合成数据生成  
- 长上下文处理  
- 记忆架构创新  

**团队愿景**  
正如Jack Morris所述："构建每个人专属的模型，实现每分钟更新，让AI随时间推移不断学习、记住细节、适应环境并像真人一样交互"。

### **📌 补充细节**
- **技术挑战**：需证明持续学习从研究问题走向稳定产品，确保模型可靠、可控、可审计  
- **差异化路径**：不同于RAG或长上下文方案，Engram押注"预学习私有上下文"的新scaling方向  
- **团队动态**：核心成员Jack Morris于2025年底放弃博士学业，专职投身Engram创业
